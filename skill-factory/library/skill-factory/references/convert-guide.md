# convert-guide.md
# 스킬 변환·통합 가이드

MERGE, CONVERT, LITE 모드에서 참조한다.

---

## §1. MERGE 모드 — 스킬 통합 기준

### 통합 판단 기준

| 조건 | 처리 방향 |
|---|---|
| 목적이 90% 이상 동일 | 통합 (최신 버전 기준, 구버전 흡수) |
| 목적이 70~90% 유사 | 통합 검토 (사용자 확인 후 결정) |
| 목적이 70% 미만 유사 | 별도 유지 |
| 한쪽이 다른 쪽의 하위 기능 | 하위 기능을 상위에 흡수 |
| 환경이 다름 (GPT vs Claude) | 별도 유지 (CONVERT로 처리) |

### 통합 절차

1. **버전 비교**: 최신 버전 식별 → 기준 버전으로 설정
2. **기능 매핑**: 두 스킬의 기능 목록을 나란히 정리
3. **중복 제거**: 동일 기능은 최신/더 나은 버전으로 통일
4. **누락 보완**: 한쪽에만 있는 유효 기능은 통합본에 추가
5. **이름 결정**: 더 포괄적인 이름으로 통일
6. **구버전 처리**: 삭제 / 보관 / 흡수 중 선택

### 출력 형식

```
[통합 분석]
| 기존 스킬 | 처리 방향 | 이유 |
|---|---|---|
| [스킬A] | 기준 버전 | 더 최신, 기능 완성도 높음 |
| [스킬B] | 흡수 | [스킬A]의 하위 기능 포함 |

[통합본 이름]: [새 이름]
[구버전 처리]: 삭제 권장 / 보관 권장

[통합본 SKILL.md]
(전체 내용)
```

## §2. CONVERT 모드 — 환경 간 변환 규칙

### 변환 매트릭스

| 원본 | 대상 | 핵심 처리 |
|---|---|---|
| GPT 지침 | Claude Skill | YAML 헤더 추가, description CSO 원칙 적용, 파일 구조 추가 |
| Claude Skill | GPT 지침 | YAML 헤더 제거, 파일 참조 → 본문 인라인, GPT 운영 방식으로 조정 |
| Claude Skill | 프로젝트 지침 | 운영 규칙 중심으로 축약, 세부 절차는 제거 |
| Full MD | Lite 지침 | 핵심 역할·모드·출력·금지만 유지, 예시·설명 제거 |
| GPT 지침 | Full MD | 상세 규칙·예시 확장, 섹션 번호 추가 |

### GPT 지침 → Claude Skill 변환 체크리스트

- [ ] YAML 헤더 (`name`, `version`, `description`) 추가
- [ ] Description CSO 5대 원칙 적용 (`quality-checklist.md §1` 참조)
- [ ] 한국어·영어 트리거 모두 포함
- [ ] GPT 전용 기능 제거 (DALL-E, 브라우징, 코드 인터프리터 등)
- [ ] "파일 저장", "메모리" 등 GPT 전용 개념 → 적절히 대체 또는 제거
- [ ] References 구조 필요 여부 판단

### Claude Skill → 프로젝트 지침 변환 체크리스트

- [ ] YAML 헤더 제거
- [ ] 운영 모드 정의 유지, 세부 절차는 요약
- [ ] References 파일 내용 → 본문 핵심만 인라인
- [ ] 150줄 이하 목표
- [ ] 금지 사항 유지

## §3. LITE 모드 — 축약 기준

### 제거 대상 (Lite에서 삭제)
- 긴 사용 예시 (2개 이상)
- 반복 설명
- 세부 규칙 설명 (규칙 자체는 유지)
- 배경 설명, 철학적 내용
- 참조 파일 상세 안내

### 유지 대상 (Lite에서 필수)
- 역할 정의 (1~2줄)
- 운영 모드 목록
- 핵심 출력 구조 (대표 1개)
- 금지 사항 (전체)
- 단축 명령어 목록

### 길이 목표
- Full MD 대비 40~50% 수준
- 프로젝트 지침: 50~150줄
- 채팅 즉시 사용: 20~50줄

## §4. 버전 관리 규칙

Semantic Versioning 기준:

| 변경 유형 | 버전 업 |
|---|---|
| 전면 재설계, 목적 변경 | Major (v1.0.0 → v2.0.0) |
| 모드 추가, 구조 변경 | Minor (v1.0.0 → v1.1.0) |
| 오타 수정, 표현 개선 | Patch (v1.0.0 → v1.0.1) |

변경 시 라이브러리 `library.md`의 해당 항목 버전도 업데이트한다.
